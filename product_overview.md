# 효명고등학교 관리 시스템 — 제품 개요 (한 장짜리)

> 2026-07-24 작성. 이 문서는 "이 시스템이 무엇이고, 무엇을 일부러 안 하는지"의 기준선이다.
> 새 기능 아이디어가 생기면 이 문서에 비추어 범위 안인지 먼저 판단한다.

## 무엇인가

효명고등학교의 **Google Workspace 운영을 자동화하는 학교 전용 관리 플랫폼**.
구글 관리 콘솔에서 수작업으로 하던 반복 업무(계정 생성·정지·삭제, 클래스룸 배정·정리,
북마크 배포, 조직도 관리)를 웹 화면 몇 번의 클릭으로 처리한다.

## 누가 쓰나

| 사용자 | 하는 일 |
|---|---|
| **정보부 관리자** (수퍼어드민) | 계정 생애주기(입학·진급·전출·졸업), 사용자/그룹/OU 전체 관리, 감사 로그 확인 |
| **일반 교직원** | 셀프서비스 — 클래스룸 강제 배정, 학기말 정리, 명렬표 인쇄, 내 프로필 관리 |
| **학생** (전교생 ~1,000명) | 학생 포털 — 졸업 시 계정 삭제 안내 확인 서명 |

## 핵심 설계 원칙

1. **Google Workspace가 데이터 원본** — 계정·클래스룸·캘린더·드라이브의 실체는 전부
   Workspace에 있고, 이 시스템의 DB(Firestore)는 예약장부·로그·설정만 담는 가벼운 기록장이다.
2. **교사 본인 권한으로 실행** — 교사 리소스를 건드리는 작업은 관리자가 아닌
   **교사 본인 이메일 사칭**으로 호출한다 (책임 소재와 권한 범위를 본인에게 귀속).
3. **되돌릴 수 있게** — 삭제·보관 등 위험 작업은 유예 기간, 복원 기능, 감사 로그를 함께 설계한다.
4. **학교당 한 채** (화이트라벨) — 다른 학교가 도입하면 코드만 가져가 자기 Firebase/Vercel
   프로젝트를 새로 세운다. 한 서버에 여러 학교를 넣지 않는다.
5. **@hmh.or.kr 전용** — 학교 도메인 계정만 로그인·접근 가능. 외부 Gmail은 차단.

## 일부러 하지 않는 것 (범위 밖)

- **성적·출결·학사행정** — NEIS의 영역. 이 시스템은 Workspace 운영 도구이지 학사 시스템이 아니다.
- **학부모 대상 서비스** — 알림(카카오 알림톡, Phase 7)은 보내되, 학부모 로그인 화면은 만들지 않는다.
- **여러 학교 동시 수용** — 위 원칙 4의 화이트라벨로 대체.
- **Workspace 밖 데이터 저장** — 수업 자료·과제물 등 콘텐츠는 시스템에 저장하지 않는다.

## 기술 구성 (요약)

Next.js (Vercel 호스팅) + Firebase Auth/Firestore + Google Workspace Admin API.
자동화는 Vercel 크론이 매일 자정(KST) 생애주기 예약을 집행. 상세는 `operations_handbook.md`.
