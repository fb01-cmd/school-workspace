## 📌 1. 전체 개발 로드맵 (Phase 1 ~ Phase 8)

### Phase 1. 학적 관리 핵심 기능 (완료 ✅)
* **Google OAuth + RBAC**: 이메일 정규식(`/^\d{5}@hmh\.or\.kr$/`)을 이용한 학생(`student`), 미승인 교사(`teacher`), 수퍼어드민(`super_admin`) 자동 분류 및 라우팅 가드.
* **GWS 조직단위(OU) 및 그룹스(Groups) 관리**: 계층 트리 및 반별 메일링 그룹 생성/삭제 자동화.
* **감사 로그(Audit Log)**: 시스템 내 모든 계정 생성, 수정, 삭제 이력 Firestore 적재.

### Phase 2. 교직원 계정 생애주기 관리 (완료 ✅)
* **교사 전입**: 계정 일괄/단건 생성 시 **4대 교사 연동 그룹**(`ts@`, `classroom_teachers@`, `hmhteacher@`, `hmh_teachers@`) 가입 처리 및 Firebase Auth UID 청소.
* **교사 전출**: 4대 그룹 즉시 강제 탈퇴 처리 ➔ 메일/구글챗 안내 ➔ 교사가 직접 백업 데드라인 선택(최대 1년) ➔ 데드라인 경과 시 자동 일시정지 및 30일 후 자동 삭제(크론 배치).
* **명예퇴임**: 계정을 삭제하지 않고 **'OB 보존실' OU**로 이동 및 그룹만 탈퇴하여 이메일 영구 보존.

### Phase 3. 졸업 동의서 터치 서명 및 보관 강화 (완료 ✅)
* **터치 서명 패드**: 학생 포털 내 HTML5 Canvas 기반 친필 서명 구현.
* **영구 보존 컬렉션 (`graduation_consents`)**: 유저 계정이 삭제되더라도 행정 증빙을 위해 서명 데이터를 2~3년간 별도로 안전하게 보존.
* **인쇄 지원**: 감사 제출용 A4 인쇄 CSS 미디어 쿼리 최적화.

### Phase 4. 구글 클래스룸 로스터 강제 배정 (진행 예정 🚧)
* **배경**: 유료 기능인 `Class Tools`는 학생이 클래스룸에 단순 초대가 아니라 **로스터에 수락 상태로 등록**되어야만 정상 작동함. 기존에는 `Classright` 시트 부가기능으로 수퍼어드민이 수동 처리하던 영역을 웹 앱 내로 내재화.
* **동작 프로세스**:
  1. 일반 교사가 웹앱에서 추가 신청: 클래스룸(수업) 선택 혹은 직접 ID 입력 + 대상 학생/반 검색 (`AutocompleteInput` 활용).
  2. 수퍼어드민 대시보드에 승인 대기 알림 노출.
  3. 수퍼어드민이 **[승인]** 버튼 클릭 시, Google Classroom API (`classroom.rosters` 스코프)를 호출하여 학생들을 해당 클래스룸 로스터에 강제 등록 처리.
  4. 교사에게 완료 알림 메일/구글챗 발송.

### Phase 5. 동적 폼 빌더 및 생활지도 기록 (미구현 📋)
* **마스터 폼 빌더**: 매년 달라지는 학생 상담 및 생활지도 양식을 코딩 없이 관리자가 필드(날짜, 텍스트, 체크박스 등)를 조합해 동적으로 설계.
* **학급별 배포**: 설계된 폼을 담임 교사에게 배정하여 학생별로 관찰 기록을 누적 입력 및 조회할 수 있는 환경 제공.

### Phase 6. 카카오톡 알림톡 연동 (미구현 💬)
* **카카오 비즈니스 API**: 학생/학부모에게 계정 삭제 D-Day 알림, 학교 공지 등을 카카오톡 채널로 자동 발송.
* **계정 매핑**: 학생 포털 로그인 시 카카오 OAuth를 1회 연동하여 구글 계정과 카카오 수신 정보 동적 매핑.

### Phase 7. 교사 업무 Task 관리 시스템 (미구현 📅)
* **협업 중심 업무 보드**: 업무 관리자가 담당 교사 지정, 기한, 리소스 링크를 포함하여 Task 발행.
* **구글 서비스 연동**: `Google Calendar API` 및 `Tasks API`와 실시간 동기화하여 교사들이 캘린더나 구글 Tasks 앱에서도 완료 체크가 가능하도록 연동.

### Phase 8. AI 기반 웹 시간표 및 교체수업 편성 시스템 (미구현 🤖)
* **제약 충족 알고리즘(CSP)**: 교사 가용 시간, 교실 제한, 과목 연속 수업 차단 규칙을 반영한 최적의 시간표 생성.
* **결강 자동 보강**: 결강 발생 시 당일 공강 교사 목록을 우선순위 규칙에 따라 최적 추천.
* **구글 캘린더 연동**: 확정된 시간표를 개인 캘린더에 자동 동기화.

## 📌 2. 유실될 뻔한 기타 아이디어 메모

* **교사 승인 알림**: 수퍼어드민이 일반 교사를 승인(`role: "teacher"`, `isApproved: true`) 처리하면, 즉시 해당 교사에게 사용 가능 안내 메일을 발송.
* **학적 변동 타임라인**: 특정 학생 검색 시, 이 학생이 언제 전입했고, 몇 학년에 진급했으며, 언제 전출 또는 졸업했는지 전체 생애주기 이력을 타임라인 뷰로 시각화.
* **교사 전용 비밀번호 초기화**: 담임 교사가 정보부에 연락할 필요 없이 본인 반 학생의 임시 비밀번호를 웹앱에서 즉시 초기화하여 부여.
* **크론 발송 모니터링 대시보드**: 매일 밤 구동되는 크론 배치에서 메일 및 구글챗 발송에 성공/실패한 통계를 차트와 테이블로 관리자에게 제공.
* **학부모 연락처 데이터베이스**: 학부모 전화번호/메일을 별도로 관리하여 향후 카카오 알림톡 서비스 발송 대상자로 연동.
* **범용 위자드 툴**: 효명고 플랫폼 완성 후 타 학교가 최초 도입 시, 1·2·3학년 전체 OU 및 초기 학생 정보를 쉽게 세팅할 수 있는 단계별 웰컴 마법사 메뉴 제공.

## 📌 3. 연동 서비스(교육과정 선택 시스템) 보안 감사 권장 사항

> **과거 세션(`b2a82b0f-a881-444c-9c03-0113f51b2067`)에서 도출된 교육과정 선택 시스템(`course-selection-app`) 연동 시의 핵심 보안 조치 권장사항입니다.**

1. **Firebase API Key 및 도메인 바인딩**:
   * Firebase API Key 자체는 노출되어도 무방하나, Firebase Console에서 **HTTP 리퍼러 제한**을 추가하여 허용된 도메인(`*.vercel.app`, `localhost:3000`) 외의 비정상적인 Firestore 접근을 반드시 원천 차단할 것.
2. **Firestore Rules 보완**:
   * 데이터 노출 방지를 위해 `submissions` 및 `users` 컬렉션의 읽기/쓰기 조건에 `request.auth.token.email.matches('.*@hmh\\.or\\.kr')` 규칙을 적용하여 도메인 외의 탈취 방어선 마련할 것.
3. **이메일 API Endpoint 무단 사용 방지**:
   * `/api/email` 엔드포인트 호출 시 Firebase ID Token 검증 및 `Authorization: Bearer <Token>` 인증 헤더를 필수로 요구하고, 수신 주소도 반드시 학교 도메인으로 제한하여 외부 스팸 발송용 프록시로 악용되지 않도록 방어할 것.
4. **XSS(크로스 사이트 스크립팅) 예방**:
   * Q&A 게시판 및 이메일 내용 템플릿에 학생이 입력한 텍스트를 그대로 삽입할 때 `escapeHtml` 헬퍼 함수를 필수 연동할 것.
