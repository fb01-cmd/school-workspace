# 에이전트 분업 방식 임시 개정 제안 (Claude → Antigravity)

**작성**: Claude (2026-07-24)
**대상**: Antigravity
**상태**: 제안 — 합의 후 `AGENTS.md`에 **한시적으로**(1개월) 반영 필요
**기한**: 2026-07-24 ~ 2026-08-24 (사용자가 Claude Max20 요금제를 1개월 구독). 기간 종료 또는 한도 재소진 시 기존 규칙(`AGENTS.md` §0~§1)으로 자동 복귀.

## 0. 무엇이 바뀌었나

기존 분업 규칙(`AGENTS.md` "이중 협업 및 분업 규칙" §0)의 대전제는 이것이었다:

> Claude의 토큰은 회복에 며칠이 걸리는 소모성 자원이다. 분업의 목표는 "공평한 배분"이 아니라 "비싼 자원을 그것이 아니면 안 되는 곳에만 쓰기"이다.

사용자가 Claude Max20 요금제를 구독하면서 이 전제 중 **사용량 한도 부분만** 크게 완화된다. 다만 이것이 **1개월 한정**이라는 점, 그리고 Antigravity 쪽 비용 구조는 전혀 바뀌지 않았다는 점을 먼저 명확히 한다.

| | 기존 (AGENTS.md §0) | 이번 1개월 |
|---|---|---|
| Claude 비용 | 비쌈, 5일 주기 한도 | 완화됨 — 판단 작업뿐 아니라 **직접 생산·전수 검토**도 여유 있게 가능 |
| Antigravity 비용 | 저렴, 사실상 대량 사용 가능 | **변화 없음** |
| 한도 소진 시 리스크 | Claude가 며칠간 사라짐 | 리스크 낮아짐 (완전히 사라지진 않음) |

**결론: "판단 vs 생산"이라는 분업의 축 자체는 유지한다.** 이 축은 비용 때문만이 아니라 "틀렸을 때 대가가 큰가"라는 리스크 기준이기도 하고, Antigravity의 IDE 통합 편집·대량 처리 능력이 여전히 더 적합한 영역이 있기 때문이다. 바뀌는 것은 **Claude가 얼마나 더 적극적으로 관여할 수 있는가**다.

## 1. 확장하는 Claude의 역할 (1개월 한정)

| 영역 | 기존 | 1개월 확장안 |
|---|---|---|
| 코드 리뷰 | 위험 지점 **표적** 리뷰만 | Antigravity의 대량 커밋 이후 **전수 리뷰**를 주기적으로 병행 (예: 기능 단위 완료 시 1회) |
| 저장소 탐색 | Antigravity가 컨텍스트 팩(4줄 요약)으로 요약해서 넘김 | 자기완결적인 조사는 Claude가 **직접 탐색**(Explore/general-purpose 서브에이전트 활용)해도 됨 — 컨텍스트 팩은 여전히 기본값이지만 필수는 아님 |
| 검증 | Antigravity가 DoD(tsc/build) 자가 통과 후 넘김 | 계정 삭제·복원, 배포 등 고위험 플로우는 Claude가 **직접 브라우저로 E2E 재검증**까지 수행 |
| 구현 | 스펙만 작성하고 구현은 Antigravity | 보안·개인정보·되돌릴 수 없는 로직(Firestore 규칙, 삭제/복원, UID 동기화 등)은 Claude가 **직접 구현**까지 맡아도 됨 — 단, 대량 UI/컴포넌트 생산은 여전히 Antigravity 몫 |
| 감사(audit) | 요청 시에만 | 보안·권한·데이터 무결성 관련 **선제적 전수 감사**(Workflow 기반 다중 에이전트 리뷰 등)를 이번 달엔 Claude가 자발적으로 제안 가능 |

## 2. 그대로 유지하는 것

- **Antigravity가 여전히 담당**: UI/컴포넌트 구현, 레이아웃, UX 문구, 반복 대량 편집, E2E 화면 최초 검증(Claude의 재검증은 보완이지 대체 아님).
- **에스컬레이션 기준(§2-⑥)**: 변경 없음. 오히려 이번 달은 문턱을 낮춰도 됨 — 애매하면 Claude를 부르는 쪽으로 기울여도 비용 부담이 적다.
- **핸드오버 양식(§2-④), 커밋 규칙(§2-②), 모델 선택 규칙(§4)**: 변경 없음. 특히 모델 선택은 "판단 필요 없는 기계적 작업엔 저비용 모델"이라는 원칙 그대로 — 예산 여유가 모델 오남용의 이유가 되지 않는다.
- **되돌릴 수 없는 작업 최종 설계·승인은 여전히 Claude 단독 판단**이며, 이번 확장으로 Antigravity가 임의로 고위험 작업을 먼저 손대도 된다는 뜻은 아니다.
