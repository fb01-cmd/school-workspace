# 개인정보 데이터 목록 (Personal Data Inventory)

> 2026-07-24 작성 (Firestore 보안 규칙 설계와 동시 조사).
> 이 시스템이 **자체 DB(Firestore)에 저장하는** 개인정보의 전수 목록이다.
> Google Workspace 안의 원본 데이터(메일·드라이브·계정)는 학교–Google 간
> Workspace for Education 계약에 따르며 이 문서의 범위 밖이다.
> 새 컬렉션에 개인정보를 저장하게 되면 **이 표를 반드시 갱신**한다.

## 원칙

- 개인정보는 목적 달성에 필요한 최소한만 저장한다 (이름·학교 이메일·학번 수준).
- 주민등록번호, 전화번호, 주소, 학부모 정보는 **저장하지 않는다** (현재 저장 사례 없음).
- 접근 통제는 `firestore.rules`(도메인 게이트 + 역할 기반)로 강제한다.

## 컬렉션별 목록

| 컬렉션 | 담는 개인정보 | 목적 | 접근(규칙 기준) | 보존·파기 |
|---|---|---|---|---|
| `users` | uid, 학교 이메일, 이름, 역할, 승인상태 | 로그인·권한 판별 | 본인 + 교직원 읽기 / 변경은 수퍼어드민 | 계정 삭제 시 함께 삭제 |
| `teacher_profiles` | 교사 이메일, 이름, 부서·담당 | 교직원 조직도 | 교직원만 | 전출·퇴직 처리 시 삭제 |
| `teacher_profiles_pending` | 위와 동일(승인 대기분) | 프로필 승인 절차 | 본인 작성, 교직원 열람 | 승인/거절 즉시 삭제 |
| `graduation_tasks` | 학생 이메일, 이름, 학번, 소속 OU, 서명 여부 | 졸업 계정 정리 예약장부 | 학생은 본인 것만, 교직원 전체 | 계정 삭제 완료 후 상태만 남김 |
| `graduation_consents` | 학생 이메일, 서명(터치 서명 데이터), 제출 시각 | 계정 삭제 안내 고지 증빙 | 교직원 읽기 전용, 쓰기는 서버만 | **증빙 목적 보존** (분쟁 대비, 파기 주기 별도 결정 필요 ⚠️) |
| `transfer_out_tasks` | 학생 이메일, 정지·삭제 예정일 | 전출·자퇴 계정 정리 | 교직원만 | 삭제 집행 시 문서 제거 |
| `teacher_transfer_tasks` | 교사 이메일, 이름, 백업 기한 | 교직원 전출 계정 정리 | 당사자 + 교직원 | 삭제 집행 후 상태만 남김 |
| `audit_logs` | 작업자 이메일, 대상자 이메일, 작업 내용, 시각 | 위험 작업 감사 추적 | 교직원 읽기, 수정·삭제 불가 | **영구 보존** (감사 목적) |
| `classroom_cleanup_logs` | 교사 이메일, 코스명 | 학기말 정리 이력·복원 | 교직원 읽기, 쓰기는 서버만 | 복원 가능성 위해 보존 |
| `classroom_sync_logs`, `chrome_bookmark_logs` | 작업 교사 이메일, 대상 학급 | 작업 이력 | 교직원 읽기 | 보존 (용량 문제 시 정리) |
| `settings` | 개인정보 없음 (OU 경로·일정 설정) | 도메인 설정 | 구성원 읽기, 교직원 쓰기 | 해당 없음 |

## 열려 있는 결정 사항

- ⚠️ `graduation_consents`(서명 증빙)의 **파기 주기** — 졸업 후 몇 년 보존할지 학교 방침으로
  정해야 함 (예: 민원 시효를 고려해 3년 보존 후 파기). 정해지면 크론에 자동 파기 추가 권장.
- `audit_logs` 영구 보존이 적정한지 — 감사 로그에도 이메일이 포함되므로, 예컨대
  "5년 경과분 자동 파기" 같은 상한을 두는 것을 검토.
