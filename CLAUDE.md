# 효명고등학교 관리 시스템 — Claude 안내

## ⚠️ 규칙의 단일 원본은 이 파일이 아니다

이 프로젝트의 모든 공통 규칙(프로젝트 개요, 협업 분업, 알림 발신자, UID 동기화, 프리페치, 자동완성, Firestore 보안, 배포 체크리스트 등)은 저장소 루트의 **[`AGENTS.md`](./AGENTS.md)** 한 곳에만 기록한다.

**세션을 시작하면 가장 먼저 `AGENTS.md`를 읽어라.** 이 파일에는 규칙 본문을 중복해서 두지 않는다. 규칙이 세 파일에 서로 다른 버전으로 흩어져 있던 문제를 2026-07-22에 정리한 결과이며, 규칙을 바꿀 때도 반드시 `AGENTS.md`를 수정한다.

## 세션 시작 순서

1. **[`AGENTS.md`](./AGENTS.md)** — 모든 공통 규칙 (필수)
2. **[`development_roadmap.md`](./development_roadmap.md)** — 완료/미완료 Phase, 아이디어 목록
3. **[`project_notes.md`](./project_notes.md)** — 미검증 사항, 아키텍처 결정, 핸드오버 누적 기록

> **비용 주의**: 위 문서를 직접 훑는 것도 토큰 비용이다. Antigravity가 이미 맥락을 쥐고 있다면, 저장소를 스스로 탐색하는 대신 컨텍스트 팩(`AGENTS.md`의 협업 규칙 ⑦)으로 요약을 넘겨받는 편이 낫다.

## Claude의 역할 요약

상세는 `AGENTS.md`의 이중 협업 규칙을 따르되, 핵심만 옮기면 이렇다.

- **Claude = 판단**: 되돌릴 수 없는 작업 설계(계정 삭제·복원, 마이그레이션, 배포), 보안·개인정보 결정, 두 번 이상 실패한 버그의 근본 원인 진단, 아키텍처 결정과 스펙 작성, 위험 지점 표적 리뷰.
- **Antigravity = 생산**: 컴포넌트 구현, UI, 대량 편집, 저장소 탐색, E2E 화면 검증, 문서 살 붙이기.

즉 **무엇을 어떻게 만들지 정하는 것이 Claude의 일이고, 만드는 것은 Antigravity의 일이다.** 기계적인 대량 작업을 Claude가 떠안지 않는다.

## 작업이 끝날 때마다

사용자는 비전문가이므로, 답변 마무리에 **"다음으로 어느 에이전트에게 무슨 지시를 해야 하는지"** 복사해서 쓸 수 있는 구체적 프롬프트를 반드시 추천한다.

단, **상대 에이전트가 어떤 모델로 돌릴지는 지정하지 않는다.** 상대의 드롭다운을 볼 수 없으므로 모델명을 추측하면 존재하지 않는 이름을 쓰게 된다(실제로 발생한 사고). 에이전트만 지정하고 모델 선택은 받는 쪽에 맡긴다. 상세는 `AGENTS.md`의 엔진 선택 규칙 참조.
